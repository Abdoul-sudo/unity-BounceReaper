# Game Brief: Bounce Reaper

**Date:** 2026-03-09
**Author:** Abdoul
**Status:** Complete

---

## Executive Summary

**Bounce Reaper** est un idle bouncer mobile où des balles auto-tuent des vagues d'ennemis dans une arène néon. More balls, more chaos, more power.

**Target Audience :** Joueurs casual à mid-core, 12-35 ans, sessions courtes sur mobile

**Core Pillars :** Satisfaction visuelle > Progression addictive > Accessible mais profond

**Key Differentiators :** Chaos visuel néon scalable, build system profond (balles + sorts + reliques), zéro skill-wall, made for TikTok/Reels content

**Platform :** Android (primary), iOS (conditionnel MacBook)

**Success Vision :** Le premier idle bouncer qui combine chaos visuel néon, builds profonds style Cat Hero, et zéro frustration — conçu pour être aussi fun à regarder qu'à jouer

---

## Game Vision

### Core Concept

An idle bouncer where balls auto-kill waves of enemies. More balls, more chaos, more power.

### Elevator Pitch

Bounce Reaper is a mobile idle bouncer where your balls auto-kill waves of enemies and drop loot. Buy more balls, upgrade them, and watch the neon chaos scale from one lonely bounce to an unstoppable swarm. More balls, more chaos, more power.

### Vision Statement

Pure visual dopamine in your pocket. Launch it, watch balls destroy everything, every session feels rewarding, come back to more.

## Target Market

### Primary Audience

Joueurs casual à mid-core, 12-35 ans, qui cherchent du fun rapide et de la satisfaction visuelle sur mobile.

**Demographics :**
- Âge : 12-35 ans
- Genre : Mixte (légère majorité masculine ~60/40)
- Plateformes : iOS / Android
- Sessions : 3-5 min, 8-15 ouvertures/jour (transports, pauses, file d'attente)

**Motivations :**
- Dopamine visuelle — chaos satisfaisant à l'écran

### Secondary Audience

- Joueurs idle/clicker hardcore (25-40 ans) qui cherchent un nouveau fix avec plus de polish visuel
- Joueurs hypercasual (12-16 ans) attirés par les pubs "satisfying chaos" sur TikTok/Reels

## Game Fundamentals

### Core Gameplay Pillars

1. **Satisfaction visuelle** — Le chaos néon de balles qui rebondissent et détruisent tout EST le jeu. Si c'est pas beau et satisfaisant, on change.
2. **Progression addictive** — Toujours un prochain unlock à portée. Le joueur ne doit jamais sentir qu'il a "fini". Layers : déblocage → étoiles → fusions → prestige.
3. **Accessible mais profond** — Idle par défaut, zéro skill-wall. Mais le joueur qui optimise ses builds (balles + sorts + reliques) est récompensé.

**Priorité des piliers :** Satisfaction visuelle > Progression addictive > Accessible mais profond

### Primary Mechanics

- **Bounce** — Les balles rebondissent automatiquement dans l'arène et tuent les ennemis au contact
- **Upgrade** — Acheter plus de balles, améliorer dégâts/vitesse/taille, débloquer nouveaux types
- **Equip** — Composer son build via le loadout : balles (3-5 slots) + sorts actifs (3 slots) + reliques passives (3-5 slots)
- **Prestige** — Reset les vagues, gagner des Souls, renforcer ses bonus permanents
- **Collect** — Ouvrir des coffres (Shards/Souls), obtenir reliques et skins

**Core Loop :**
```
Balles rebondissent → Tuent ennemis → Drop Shards + XP
  → Upgrade balles / Ouvre coffres → Obtient reliques + sorts
    → Compose un build → Vagues plus fortes
      → Prestige → Souls → Bonus permanents → Recommence plus fort
```

### Player Experience Goals

- **Dopamine visuelle** — Chaque session est un spectacle de chaos néon
- **Power fantasy** — De 1 balle à 30 balles godlike, le joueur se sent tout-puissant
- **Optimisation satisfaisante** — Tester des combos balles/sorts/reliques et voir les résultats
- **Récompense constante** — Jamais plus de 30 secondes sans un drop, un unlock ou un upgrade

**Emotional Journey :** "Oh une balle" → "Oh 10 balles" → "OH 30 BALLES QUI BRÛLENT TOUT" → "Je suis un dieu" → Prestige → "Recommençons, cette fois en build AoE"

### Scaling Design

- Cap visuel à ~30 balles — au-delà, les upgrades augmentent la puissance, pas le nombre
- Sorts : déblocage → étoiles ★1-★5 → fusions → variantes
- Le nombre plafonne, la puissance jamais

### Build System

| Layer | Slots | Exemples |
|-------|-------|----------|
| Balles équipées | 3-5 | Feu, Glace, Explosive |
| Sorts actifs | 3 | Multiball, Berserk, Gravity Well |
| Reliques (passifs) | 3-5 | +30% DoT, +15% vitesse, heal on kill |
| Skins (cosmétique + buff mineur) | Balle + Arène | +8% vitesse, +15% Shards |

### Monnaies

| Monnaie | Type | Obtention | Utilisation |
|---------|------|-----------|-------------|
| **Shards** (éclats) | Soft | Gameplay, idle, stages | Upgrades, coffres d'or |
| **Souls** (âmes) | Hard | Lentement en jouant, IAP, prestige | Coffres premium, balles spéciales, accélération |

### Coffres

| Coffre | Prix | Contenu |
|--------|------|---------|
| Coffre de Shards | 500 Shards | 1 relique commune/rare, Shards, XP |
| Coffre de Souls | 50 Souls | 1 relique rare/épique garantie, skin fragment |
| Coffre Légendaire | 300 Souls | 1 relique épique/légendaire + balle spéciale |

## Scope and Constraints

### Target Platforms

**Primary :** Android (Google Play) — 25$ one-time, build direct depuis Windows
**Secondary :** iOS (App Store) — conditionnel à l'achat d'un MacBook Pro (M4/M5 Pro envisagé) + 99$/an Apple Developer

### Budget Considerations

- **Budget total :** ~0-100$
- **Assets achetés :** DOTween Pro, All In 1 Sprite Shader, Text Animator, vHierarchy, Hot Reload, Ultimate Game Music Collection — déjà en possession
- **Assets graphiques :** Gratuits (OpenGameArt, itch.io, Kenney.nl) ou générés procéduralement (formes géométriques néon = faisable sans artiste)
- **Marketing :** 0$ — acquisition organique (TikTok/Reels gameplay, ASO)
- **Serveurs :** Google Play Games Services (gratuit), Firebase (free tier)

### Team Resources

- **Équipe :** Solo dev (Abdoul) + Claude Code (AI pair-programmer)
- **Disponibilité :** Side-project (temps partiel)
- **Skills :** Bases en dev, utilisation d'assets existants
- **Claude Code :** Rôle principal en coding, architecture, debugging
- **Matériel :** ASUS ROG Strix G17 (Windows), MacBook Pro M4/M5 Pro envisagé

**Skill Gaps :**

| Gap | Mitigation |
|-----|------------|
| Art/sprites | Style géométrique néon = formes simples + shaders (All In 1 Sprite Shader). Pas besoin d'artiste |
| Sound design | Ultimate Game Music Collection + SFX gratuits (freesound.org) |
| Marketing | Organique uniquement — gameplay néon = contenu viral naturel |
| Backend | Firebase free tier + Google Play Games Services |

### Technical Constraints

- **Engine :** Unity 6.3 LTS (6000.3.10f1), 2D URP
- **Performance :** 60fps par défaut, option 120fps sur appareils compatibles
- **Appareils min :** Android 8+ / iOS 14+ (milieu de gamme 2022+)
- **Taille app :** < 150MB (important pour conversion mobile)
- **Online :** Google Play Games (leaderboards, achievements), Cloud Save via Google/Firebase
- **Object pooling :** Obligatoire dès le jour 1 (30 balles + ennemis + particules)

### Scope Realities

- Le style géométrique néon est un **avantage scope** — pas besoin d'art complexe, les shaders font le travail
- Claude Code compense largement le gap technique — architecture, code, debugging
- Le MVP doit prouver le core loop AVANT d'ajouter la monétisation complète
- Le side-project impose un développement incrémental — fonctionnalité par fonctionnalité

## Reference Framework

### Inspiration Games

**Archero**
- Taking : Progression par skills, loop session courte, coffres, système de monétisation (ads rewarded, battle pass, offres contextuelles)
- Not Taking : Le skill-wall, le contrôle direct du personnage, la difficulté punitive

**Cat Hero**
- Taking : Système de prestige multi-layer, builds via loadout, profondeur long-terme
- Not Taking : Le style graphique, la complexité UI

**Idle Slayer**
- Taking : Le idle pur qui farm tout seul, le prestige qui reset, la satisfaction passive
- Not Taking : Les visuels datés, le manque de chaos visuel

**Blade Ball (Roblox)**
- Taking : La viralité du concept "balles", le côté spectaculaire
- Not Taking : Le gameplay PvP, le skill-based, Roblox

**Hole.io**
- Taking : La satisfaction de grossir/tout absorber, le onboarding 5 secondes
- Not Taking : Le gameplay non-idle, le manque de progression long-terme

### Competitive Analysis

**Direct Competitors :**

| Concurrent | Forces | Faiblesses |
|-----------|--------|-----------|
| Archero | Monétisation excellente, loop addictive | Frustrant, skill-wall, pas idle |
| Idle Slayer | Bon idle, prestige satisfaisant | Visuels faibles, pas de chaos visuel |
| Cat Hero | Builds profonds, prestige multi-layer | UI complexe, pas de bouncer |
| Blade Ball | Viral, spectaculaire | Pas idle, pas de progression |

**Marché :** Espace idle mobile très actif mais aucun idle bouncer néon avec builds profonds. Le créneau est ouvert.

### Key Differentiators

1. **Chaos visuel néon scalable** — Le jeu EST le spectacle. Aucun idle bouncer ne mise autant sur la dopamine visuelle
2. **Build system profond** (balles + sorts + reliques) — Rare dans les idle games, inspiré Cat Hero
3. **Zéro skill-wall** — Contrairement à Archero, le joueur n'est jamais bloqué par manque de skill
4. **Made for content** — Le gameplay néon est conçu pour TikTok/Reels. Marketing intégré au design

**Unique Value Proposition :**
Le premier idle bouncer qui combine chaos visuel néon, builds profonds style Cat Hero, et zéro frustration — conçu pour être aussi fun à regarder qu'à jouer.

## Content Framework

### World and Setting

**"The Void Arena"** — Un espace entre les mondes où le Reaper récolte des Souls. Pas de lore complexe, le thème justifie tout :
- Fond sombre → le Void
- Néon → l'énergie des âmes
- Ennemis qui spawn → ils traversent le Void
- "Reaper" → récolte de Souls
- Prestige → transcendance

### Narrative Approach

**Minimal.** Zéro dialogue, zéro cutscene. Le gameplay parle tout seul.
- Noms thématiques pour les boss (Void Guardian, Soul Devourer, etc.)
- Descriptions d'1 ligne pour les reliques et sorts
- Le prestige est une "transcendance" narrative

**Story Delivery :** Purement environnemental — noms, visuels, ambiance

### Content Volume

**MVP Semaine 1 (STRICT) :**
- Balle qui rebondit (physique 2D)
- Ennemis qui spawn par vagues (3-4 types : Blob, Bat, Skeleton, Shield Knight)
- Kill → drop Shards + particules + feedback
- Upgrades : dégâts, vitesse, +1 balle
- Boss toutes les 10 vagues (timer 30-60s, "Boss escaped!" si fail)
- Sauvegarde locale (JSON)
- UI basique (HUD + shop upgrades)
- Game feel : glow, trails, screen shake, damage numbers

**Semaine 2 :** Prestige basique, types de balles supplémentaires (Feu, Glace)
**Semaine 3 :** Sorts actifs (Multiball, Berserk, Gravity Well) + reliques + coffres
**Semaine 4+ :** Build system complet, monétisation, skins, events, leaderboards, cloud save

### Fail System

| Situation | Résultat |
|-----------|----------|
| Vagues normales | Balles immortelles, idle pur, zéro mort possible |
| Boss (toutes les 10 vagues) | Timer 30-60s. Fail → "Boss escaped!" |
| Fail boss | Reste à la vague actuelle, farm, upgrade, retente |
| Revive boss | Pub rewarded → relance avec HP restants du boss |
| Bloqué longtemps | Offre contextuelle "Pack Force" |

## Art and Audio Direction

### Visual Style

**Style géométrique néon sur fond sombre — 0$ d'assets custom nécessaires**

| Élément | Solution | Coût |
|---------|----------|------|
| Balles | Sprite cercle Unity + shader glow (All In 1) | 0$ |
| Ennemis | Formes géométriques (carré, triangle, hexagone) + couleurs | 0$ |
| Arène | Fond noir + bordures néon (LineRenderer) | 0$ |
| Particules | Unity Particle System built-in | 0$ |
| UI | TextMeshPro + DOTween animations | 0$ |
| Trails | Trail Renderer component | 0$ |
| Icônes | Kenney.nl game icons (gratuit, CC0) | 0$ |
| Boss | Mêmes formes mais plus grosses + effets spéciaux | 0$ |

**Palette néon :** Cyan, magenta, jaune néon, vert néon sur fond noir/gris foncé. Max 3-4 couleurs simultanées.

**Game feel recette :**
- Bloom/glow URP Post Processing sur tout
- Particules à chaque kill
- Screen shake léger (DOTween) aux impacts
- Damage numbers animés (Text Animator)
- Trail Renderer sur chaque balle
- Option "reduced effects" dans les settings

### Audio Style

- **Musique :** 2 tracks synthwave/électro (Ultimate Game Music Collection) — ambiance + boss fight
- **SFX :** ~15 sons — rebond (pling cristallin, pitch variable), kill (pop), coin pickup, sort, boss, UI
- **Sources SFX :** freesound.org (gratuit)
- **Voice acting :** Aucun

### Production Approach

- Formes géométriques + shaders = pas besoin d'artiste
- All In 1 Sprite Shader fait 90% du travail visuel
- Assets gratuits uniquement (Kenney.nl, freesound.org)
- Claude Code pour le code, l'architecture et le debugging
- Architecture data-driven (ScriptableObjects) dès le jour 1

| Discoverability | ASO solide, capturer du gameplay TikTok dès le prototype, soft launch |

## Success Criteria

### MVP Definition

**MVP Semaine 1 (STRICT) — Le core loop doit être fun :**
- Balle qui rebondit (physique 2D)
- Ennemis qui spawn par vagues (3-4 types)
- Kill → drop Shards + particules + feedback
